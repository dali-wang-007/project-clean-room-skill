# 项目净化与接管 Skill

`project-clean-room` 是一个中文 AI 协作 Skill，用来解决长期任务里“对话越来越长、新窗口接不上、同事换工具后还要重新解释一遍”的问题。

它不绑定任何具体业务项目，也不包含任何私有资料。你可以把它用于正式项目，也可以用于一个比较大的任务、专题、方案、PRD、知识库、竞品分析或长期文档输出。

## 一句话说明

每个项目或大任务维护一份《Agent团队协作文件》；当前窗口输入“净化”，AI 就整理并更新这份文件；新窗口输入“接管”，AI 就先学习这份文件，再继续配合你工作。

## 先给产品同事看的版本

你不需要先学开发，也不一定要安装到本机目录。

最简单的用法是：

1. 复制下方“对话级启用提示词”；
2. 粘贴到 WorkBuddy、Cursor、ChatGPT、Codex 或其他 AI 工具的新对话里；
3. 再把你的《Agent团队协作文件》发给 AI，或让 AI 在当前项目文件夹里查找；
4. 后续你只要输入“净化”“接管”“/净化”或“/接管”，AI 就按这个 Skill 的规则执行。

这相当于在当前对话里临时启用 Skill。它不是永久安装，但对大部分产品经理、运营、业务同事来说已经够用了。

这里有一个很重要的区别：

| 方式 | 适合谁 | 你实际要做什么 | 它会发生什么 |
|---|---|---|---|
| 复制“对话级启用提示词” | 产品经理、业务、运营、不会安装 Skill 的同事 | 把提示词粘到自己的 AI 对话框里 | 只在当前对话临时启用这套规则，不会生成新 Skill，也不是永久安装 |
| 复制 GitHub 仓库链接 | 开发、会配置工具的同事、需要长期复用的人 | 下载或安装仓库里的 `project-clean-room` 文件夹 | 把 Skill 放进工具支持的 Skill 目录或规则配置里，后续可长期复用 |

所以，不要理解成“把这段提示词发给 AI，让 AI 自己生成一个 Skill”。这段提示词本身就是一份压缩版执行规则，用来让不支持安装 Skill 的工具也能临时按同一套逻辑工作。

也不要理解成“只复制 GitHub 链接就能自动使用”。如果工具不能自动读取和安装 GitHub 仓库，光发链接通常不够，还需要复制提示词，或者由开发同事按仓库说明安装。

## 为什么 `/` 菜单里查不到

有些 AI 工具的 Skill 不会自动变成 `/` 菜单命令。安装成功后，工具可能只是把 Skill 放进后台可检索规则里，由用户输入的自然语言触发。

所以 `/` 查不到不一定是安装失败。你可以直接输入下面任意一种：

| 输入方式 | 是否支持 | 说明 |
|---|---|---|
| 净化 | 支持 | 推荐给小白同事，最稳 |
| 接管 | 支持 | 推荐给小白同事，最稳 |
| /净化 | 支持 | 适合习惯斜杠命令的人，即使 `/` 菜单没有显示也可以直接发 |
| /接管 | 支持 | 适合习惯斜杠命令的人，即使 `/` 菜单没有显示也可以直接发 |
| $project-clean-room 净化 | 支持 | 适合支持 `$skill-name` 显式调用的工具 |
| $project-clean-room 接管 | 支持 | 适合支持 `$skill-name` 显式调用的工具 |

结论：能在 `/` 菜单里找到当然可以点；找不到也没关系，直接把 `净化`、`接管`、`/净化`、`/接管`、`$project-clean-room 净化` 或 `$project-clean-room 接管` 发出去即可。

需要注意：是否能在 `/` 菜单中搜索到，取决于具体 AI 工具的产品机制。这个仓库已经补充了 `agents/openai.yaml`，用于给支持 Skill UI 展示的工具提供名称、简介和默认提示词，但它仍不能强制所有工具把 Skill 显示成 `/净化` 或 `/接管` 菜单项。

## 对话级启用提示词

把下面这段完整复制到你的 AI 对话框里，即可在当前对话中启用本 Skill：

```text
从现在开始，请在本次对话中启用“项目净化与接管 Skill”。

这个 Skill 用于长期项目、大任务、专题或文档协作的上下文整理和跨窗口接续。这里的“项目”是广义概念，不一定是公司正式立项的业务项目，也可以是我正在处理的一个较大的任务、方案、PRD、知识库、汇报材料、研究专题或长期输出事项。

核心文件叫《Agent团队协作文件》。它是给我、团队成员和 AI 一起看的协作入口，用来记录当前有效背景、目标范围、关键规则、输出偏好、已产出资料、下一步工作和待确认问题。

触发规则：
1. 只要我的消息中包含“净化”或“/净化”，你就执行净化流程。
2. 只要我的消息中包含“接管”或“/接管”，你就执行接管流程。
3. 如果工具支持 `$skill-name` 显式调用，我输入 `$project-clean-room 净化` 或 `$project-clean-room 接管` 也应触发。
4. 如果工具的 `/` 菜单里查不到这个 Skill，也不影响执行；我直接输入“净化”“接管”“/净化”“/接管”“$project-clean-room 净化”或“$project-clean-room 接管”都应触发。

净化流程：
1. 先查找当前项目或任务对应的《Agent团队协作文件》。
2. 如果已有文件，先读取并以它为准更新；如果没有，就根据我提供的资料生成一份。
3. 净化结果需要先告诉我：你找到或生成了哪份《Agent团队协作文件》，它放在哪里，主要沉淀了什么，后续当前窗口如何继续协作，其他窗口如何接管。
4. 净化时只沉淀当前仍然有效的信息，不要把长篇聊天记录原样复制进去。

接管流程：
1. 先学习我提供的或当前项目文件夹中的《Agent团队协作文件》。
2. 学习后输出你对项目或任务的理解，包括：目标、范围、关键规则、输出偏好、已产出资料、未来需要协助我的工作项、仍需确认的问题。
3. 最后必须问我：这是我学习《Agent团队协作文件》内容后，对项目内容的理解，以及我未来需要配合你的工作项。你确认我理解得是否正确？
4. 在我确认前，不要直接大规模改写正式资料。

正式文档输出要求：
1. 不在正式文档正文里展示输出时间、创建时间、生成时间等元信息。
2. 不把 AI 的工作说明、提示词说明、版本融合过程放进正式文档。
3. 除非我明确要求技术文档，否则不要在正式文档中展示代码块。
4. 优先使用产品经理能直接交付的结构，例如目标、范围、角色、流程、规则、字段、异常、指标、验收、待确认问题。
5. 如果我的说法可能不准确，你要先做判断，再提醒我，而不是直接顺着我说。
```

仓库中也单独提供了可复制版本：`project-clean-room/references/chat-install-prompt.md`。

## 名词解释

| 名词 | 小白解释 |
|---|---|
| Skill | 一套让 AI 按固定方式做事的说明书。它不是业务资料本身，而是告诉 AI 遇到某类任务时应该怎么处理。 |
| 项目 | 这里是广义说法，不一定是公司正式立项的项目，也可以是一个需要持续推进的大任务、专题、方案、文档包或研究事项。判断标准是：资料会不断变化、需要多轮 AI 协作、可能要跨窗口或多人接续。 |
| Agent团队协作文件 | 给人和 AI 一起看的协作入口文件。它记录当前有效的背景、目标、范围、规则、输出偏好、资料位置、下一步工作和待确认问题。 |
| 净化 | 当前对话变长、变慢或信息混乱时，让 AI 把有效内容整理进《Agent团队协作文件》，后续以这份文件为准继续工作。 |
| 接管 | 新窗口、新工具或新同事加入时，让 AI 先学习《Agent团队协作文件》，再输出理解总结并请你确认。 |
| 对话级启用 | 不安装文件，只把提示词复制到聊天窗口里，让 AI 在本次对话按 Skill 规则工作。适合 WorkBuddy、ChatGPT 和不方便安装插件的场景。 |
| 本地安装 | 把 Skill 文件夹放到工具指定目录，让工具长期识别这个 Skill。适合 Codex 或支持自定义 Skill 的工具。 |

## 什么时候用

| 场景 | 你可以怎么说 | AI 应该做什么 |
|---|---|---|
| 当前对话太长，回答变慢 | 净化、/净化 | 查找或生成《Agent团队协作文件》，更新有效信息，并解释产出物 |
| 上下文被压缩后继续工作 | 净化一下、/净化 | 以《Agent团队协作文件》为准整理当前协作状态 |
| 新开窗口继续同一件事 | 接管、/接管 | 先学习《Agent团队协作文件》，再输出理解总结并请你确认 |
| 换到 Cursor 或 WorkBuddy 继续 | 接管 | 读取你提供的协作文件，恢复项目背景和输出偏好 |
| 新同事加入 | 接管这个项目 | 用协作文件快速了解背景、规则、资料和下一步工作 |

## WorkBuddy 使用方法

WorkBuddy 通常更适合“对话级启用”。

1. 新开一个对话。
2. 复制上面的“对话级启用提示词”并发送。
3. 上传或粘贴你的《Agent团队协作文件》。
4. 如果你还没有协作文件，就让 AI 先根据现有资料生成一份。
5. 后续需要整理上下文时输入“净化”或“/净化”。
6. 新窗口继续时输入“接管”或“/接管”，再提供同一份《Agent团队协作文件》。

推荐给同事的最短说法：

```text
先复制 README 里的“对话级启用提示词”到 WorkBuddy，再上传《Agent团队协作文件》。以后当前窗口乱了说“净化”或“/净化”，新窗口继续说“接管”或“/接管”。
```

## Cursor 使用方法

Cursor 可以按两种方式用。

### 小白方式：Chat + 文件引用

1. 把 `agent-team-collaboration-template.md` 复制到你的项目目录。
2. 改名为 `Agent团队协作文件.md`。
3. 在 Cursor Chat 里先粘贴“对话级启用提示词”。
4. 输入：

```text
接管。请先读取 @Agent团队协作文件.md。
```

如果 Cursor 没有正确读取文件，就把文件正文直接复制到聊天框。

### 进阶方式：Project Rule

如果团队熟悉 Cursor Rules，可以把 `project-clean-room/SKILL.md` 的核心规则放入项目规则：

- 用户消息包含“净化”时，执行净化流程；
- 用户消息包含“/净化”时，也执行净化流程；
- 用户消息包含“接管”时，执行接管流程；
- 用户消息包含“/接管”时，也执行接管流程；
- 执行前优先读取《Agent团队协作文件》；
- 重要产出后提醒更新《Agent团队协作文件》；
- 正式文档不展示输出时间、创建时间、生成时间等元信息。

## ChatGPT 使用方法

1. 新开一个聊天。
2. 复制“对话级启用提示词”并发送。
3. 上传或粘贴《Agent团队协作文件》。
4. 输入“接管”，让 ChatGPT 先输出理解总结。
5. 你确认理解正确后，再让它继续写 PRD、方案、汇报、测试用例等内容。

## Codex 本地安装方法

如果你使用 Codex，并希望它长期识别这个 Skill，可以安装到本机 Skill 目录。

```bash
cp -R project-clean-room ~/.codex/skills/
```

如果当前会话没有自动加载，重新打开一个对话窗口即可。

## 给开发看的版本

### 仓库结构

```text
project-clean-room-skill/
├── README.md
├── CONTRIBUTING.md
├── LICENSE
└── project-clean-room/
    ├── agents/
    │   └── openai.yaml
    ├── SKILL.md
    └── references/
        ├── agent-team-collaboration-template.md
        └── chat-install-prompt.md
```

### 文件说明

| 文件 | 作用 |
|---|---|
| `project-clean-room/SKILL.md` | Skill 本体，给支持 Skill 的工具读取，定义触发条件、执行流程和输出规则 |
| `project-clean-room/agents/openai.yaml` | UI 元信息，给支持 Skill 展示的工具提供名称、简介和默认提示词 |
| `project-clean-room/references/agent-team-collaboration-template.md` | 《Agent团队协作文件》模板，复制到具体项目或任务目录后填写 |
| `project-clean-room/references/chat-install-prompt.md` | 对话级启用提示词，给不支持安装 Skill 的工具使用 |
| `README.md` | 给团队成员看的使用说明 |
| `CONTRIBUTING.md` | 维护和提交规范 |

### 维护原则

- Skill 仓库只放通用规则，不放任何具体项目私有上下文。
- 具体项目资料应放在该项目自己的《Agent团队协作文件》中。
- `SKILL.md` 保持偏工具执行规则，`README.md` 保持偏用户使用说明。
- 面向产品经理的说明要尽量用“小白能复制粘贴”的方式表达。

## 常见问题

| 问题 | 回答 |
|---|---|
| 我把提示词复制到聊天框，是不是就永久安装了 Skill？ | 不是。它是“对话级启用”，只对当前聊天有效。但对 WorkBuddy、ChatGPT 这类工具已经够用。 |
| 我能不能把这个 Skill 用到其他项目？ | 可以。Skill 是通用规则；但每个项目或任务都应该有自己的《Agent团队协作文件》。 |
| 《Agent团队协作文件》是不是要分享给所有同事？ | 只分享对应项目或任务需要共享的协作文件。不要把敏感资料放进公开仓库。 |
| 没有正式项目，也能用吗？ | 可以。只要是一个需要多轮 AI 协作、资料持续变化、可能跨窗口继续的大任务，就可以用。 |
| 和系统自动压缩上下文有什么区别？ | 系统压缩是平台自动做的，不一定知道哪些信息最重要；本 Skill 是主动整理有效信息，并形成可给其他窗口和其他工具读取的协作文件。 |
