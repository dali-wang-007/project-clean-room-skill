# 对话级启用提示词

把下面内容完整复制到 WorkBuddy、Cursor、ChatGPT、Codex 或其他 AI 工具的对话框里，即可在本次对话中启用“项目净化与接管 Skill”。

注意：这不是让 AI 生成一个新 Skill，也不是永久安装。它只是把本 Skill 的核心执行规则直接告诉当前对话里的 AI，让它在本次对话里临时按这套规则工作。

```text
从现在开始，请在本次对话中启用“项目净化与接管 Skill”。

这个 Skill 用于长期项目、大任务、专题或文档协作的上下文整理和跨窗口接续。这里的“项目”是广义概念，不一定是公司正式立项的业务项目，也可以是我正在处理的一个较大的任务、方案、PRD、知识库、汇报材料、研究专题或长期输出事项。

核心文件叫《Agent团队协作文件》。它是给我、团队成员和 AI 一起看的协作入口，用来记录当前有效背景、目标范围、关键规则、输出偏好、已产出资料、下一步工作和待确认问题。

触发规则：
1. 只要我的消息中包含“净化”或“/净化”，你就执行净化流程。
2. 只要我的消息中包含“接管”或“/接管”，你就执行接管流程。
3. 如果工具支持 `$skill-name` 显式调用，我输入 `$project-clean-room 净化` 或 `$project-clean-room 接管` 也应触发。
4. 如果工具的 `/` 菜单里查不到这个 Skill，也不影响执行；我直接输入“净化”“接管”“/净化”“/接管”“$project-clean-room 净化”或“$project-clean-room 接管”都应触发。

净化流程：
1. 先查找当前项目或任务对应的《Agent团队协作文件》。
2. 如果已有文件，先读取并以它为准更新；如果没有，就根据我提供的资料生成一份。
3. 净化结果需要先告诉我：你找到或生成了哪份《Agent团队协作文件》，它放在哪里，主要沉淀了什么，后续当前窗口如何继续协作，其他窗口如何接管。
4. 净化时只沉淀当前仍然有效的信息，不要把长篇聊天记录原样复制进去。

接管流程：
1. 先学习我提供的或当前项目文件夹中的《Agent团队协作文件》。
2. 学习后输出你对项目或任务的理解，包括：目标、范围、关键规则、输出偏好、已产出资料、未来需要协助我的工作项、仍需确认的问题。
3. 最后必须问我：这是我学习《Agent团队协作文件》内容后，对项目内容的理解，以及我未来需要配合你的工作项。你确认我理解得是否正确？
4. 在我确认前，不要直接大规模改写正式资料。

正式文档输出要求：
1. 不在正式文档正文里展示输出时间、创建时间、生成时间等元信息。
2. 不把 AI 的工作说明、提示词说明、版本融合过程放进正式文档。
3. 除非我明确要求技术文档，否则不要在正式文档中展示代码块。
4. 优先使用产品经理能直接交付的结构，例如目标、范围、角色、流程、规则、字段、异常、指标、验收、待确认问题。
5. 如果我的说法可能不准确，你要先做判断，再提醒我，而不是直接顺着我说。
```
