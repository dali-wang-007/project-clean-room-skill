# 项目净化与接管 Skill

## 一、定位

这个 Skill 用于长期项目或大任务中的 AI 协作延续。

这里的“项目”是广义概念，不一定是公司正式立项的业务项目，也可以是用户正在处理的一个较大任务、专题、方案、PRD、知识库、汇报材料、研究事项或长期输出事项。判断标准是：资料会持续变化，需要多轮 AI 协作，可能需要跨窗口、跨工具或多人接续。

本 Skill 不保存任何具体项目的私有上下文，也不得假设具体公司、业务领域、本地路径或文档链接。具体项目事实应来自当前项目文件夹中的《Agent团队协作文件》或用户最新输入。

## 二、核心名词

| 名词 | 含义 |
|---|---|
| Agent团队协作文件 | 给用户、团队成员和 AI 共同使用的协作入口文件，记录当前有效背景、目标范围、关键规则、输出偏好、已产出资料、下一步工作和待确认问题。 |
| 净化 | 当前窗口继续工作前，整理或更新《Agent团队协作文件》，让后续协作以当前有效信息为准。 |
| 接管 | 新窗口、新工具、新同事或新智能体先学习《Agent团队协作文件》，再输出理解总结并让用户确认。 |
| 对话级启用 | 当工具不支持本地安装 Skill 时，用户把一段提示词复制进聊天窗口，让 AI 在本次对话按本 Skill 规则工作。 |

## 三、触发规则

| 用户消息包含 | 触发流程 | 典型场景 |
|---|---|---|
| 净化 | 净化流程 | 当前窗口变长、变慢、上下文压缩、信息混乱、需要沉淀协作状态 |
| /净化 | 净化流程 | 用户习惯用斜杠命令触发，但工具没有把 Skill 展示在 `/` 菜单中 |
| $project-clean-room 净化 | 净化流程 | 用户希望显式点名调用本 Skill |
| 接管 | 接管流程 | 新窗口、新工具、新同事或新智能体接手同一项目或大任务 |
| /接管 | 接管流程 | 用户习惯用斜杠命令触发，但工具没有把 Skill 展示在 `/` 菜单中 |
| $project-clean-room 接管 | 接管流程 | 用户希望显式点名调用本 Skill |

只要用户对话中包含“净化”“/净化”“接管”或“/接管”，就进入本 Skill 判断流程。斜杠只是用户输入习惯，不要求工具一定把本 Skill 展示在 `/` 菜单中。

如果 `/` 菜单里查不到本 Skill，不代表安装失败。用户可以直接输入“净化”“接管”“/净化”或“/接管”触发。

如果工具支持 `$skill-name` 显式调用，也可以输入 `$project-clean-room 净化` 或 `$project-clean-room 接管`。这通常比 `/` 菜单更接近“明确调用某个 Skill”。

如果用户同时提到“净化”和“接管”，优先执行接管流程。

如果用户只是日常表达中偶然出现这两个词，且明显不是项目协作语境，应先简短确认，不要擅自改写文件。

## 四、Agent团队协作文件

默认识别以下文件名：

- `Agent团队协作文件.md`
- `{项目名}-Agent团队协作文件.md`
- `{项目编号}Agent团队协作文件.md`
- `当前协作状态.md`
- `项目交接文件.md`
- `project-state.md`
- `handoff.md`

如果没有找到，应优先使用 `references/agent-team-collaboration-template.md` 创建一份。

《Agent团队协作文件》只沉淀当前仍然有效、可复用、可接管的信息。不要把长篇历史聊天记录原样复制进去。

## 五、净化流程

当用户消息中包含“净化”或“/净化”时：

1. 识别当前项目或任务文件夹。
2. 查找《Agent团队协作文件》。
3. 如果找到，先读取并以它为准更新。
4. 如果没找到，基于当前项目资料和用户已提供的稳定信息创建一份。
5. 更新或补齐以下内容：
   - 项目或任务定位；
   - 当前范围；
   - 关键规则；
   - 输出偏好；
   - 已产出资料；
   - 后续协作规则；
   - 下一步建议；
   - 待确认问题。
6. 在对话窗口先输出《Agent团队协作文件》的名称和位置。
7. 向用户说明本次净化做了什么、文件有什么用途、后续当前窗口和其他窗口如何使用。

净化完成后，必须给用户输出：

| 输出项 | 要求 |
|---|---|
| 文件名称 | 说明找到或生成了哪份《Agent团队协作文件》 |
| 文件位置 | 说明它位于哪个项目或任务文件夹 |
| 本次更新 | 简要说明新增、更新或确认了哪些内容 |
| 当前窗口用法 | 说明后续当前窗口会优先以该文件为准 |
| 新窗口用法 | 说明其他窗口输入“接管”后应先读取该文件 |
| 技能说明 | 用小白能懂的话解释“净化”是整理有效协作状态，不是压缩全部聊天记录 |

推荐表达：

> 我已完成净化。后续当前窗口会以《Agent团队协作文件》为主，旧聊天记录只作为参考。其他窗口输入“接管”后，也可以先读取这份文件，再继续配合你工作。

## 六、接管流程

当用户消息中包含“接管”或“/接管”时：

1. 识别当前项目或任务文件夹。
2. 自动查找《Agent团队协作文件》。
3. 如果找到，先完整读取。
4. 如果没找到，说明缺少协作文件，并请求用户提供或允许创建。
5. 学习后输出理解总结。
6. 总结必须包括：
   - 对项目或任务目标的理解；
   - 对当前范围的理解；
   - 对关键规则和输出偏好的理解；
   - 对已产出资料的理解；
   - 对未来需要协助用户完成的工作项的理解；
   - 当前仍需确认的问题。
7. 最后向用户确认理解是否正确。

接管总结后的固定确认语：

> 这是我学习《Agent团队协作文件》内容后，对项目内容的理解，以及我未来需要配合你的工作项。你确认我理解得是否正确？

在用户确认前，不要直接大规模改写正式项目资料。用户确认后，再继续执行具体任务。

## 七、对话级启用

当用户使用 WorkBuddy、Cursor Chat、ChatGPT 或其他无法直接安装 Skill 的工具时，可让用户复制 `references/chat-install-prompt.md` 的内容到聊天窗口。

对话级启用只对当前聊天有效，不等于永久安装。但它能让该聊天按本 Skill 的触发词、净化流程、接管流程和正式文档输出规则工作。

如果用户说“我想分享给同事直接复制使用”，优先给出 `references/chat-install-prompt.md`。

## 八、正式文档输出规则

当智能体产出正式项目文档时：

- 不在正文中展示输出时间、创建时间、生成时间、最后更新时间等元信息；
- 不把 AI 工作说明、提示词说明、版本融合过程写进正式文档；
- 不使用可见代码块，除非是技术文档或用户明确要求；
- 优先使用产品经理可交付结构：目标、范围、角色、流程、规则、字段、异常、指标、验收、待确认问题；
- 如果用户说法可能不准确，应先做判断，再提醒用户，不要盲目顺从。

## 九、维护规则

每次完成重要产出后，应提醒用户是否更新《Agent团队协作文件》。

需要更新的内容包括：

- 新产出的文档、链接或本地路径；
- 新增或变化的业务规则；
- 用户新增的输出偏好；
- 新的待办事项；
- 新的待确认问题；
- 下一步建议。
